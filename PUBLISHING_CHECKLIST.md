# Чек‑лист подготовки MedGuide к публикации

## 1) Версии и идентификаторы
- `app.json`:
  - `expo.version`
  - `android.versionCode` (увеличивать при каждом релизе)
  - `ios.buildNumber` (увеличивать при каждом релизе)
  - `android.package` и `ios.bundleIdentifier` (не менять после публикации)

## 2) Конфигурация API
- Установить переменную окружения: **`EXPO_PUBLIC_API_URL`** (пример: `http://<server>:8000/api/`)
- Проверить авторизацию/токены/ошибки API.

## 3) Ассеты
- Иконка, adaptive icon, splash (проверить отображение на разных устройствах).

## 4) Разрешения
- Камера (для QR).
- Интернет (для работы с сервером).

## 5) Тестирование перед релизом
- `npm test`
- Пройти основные сценарии: регистрация/вход, список препаратов, выдача, профиль, восстановление пароля.

## 6) Документы и метаданные
- `PRIVACY_POLICY.md` (разместить публично и указать ссылку в магазине)
- `STORE_METADATA.md`
- Скриншоты приложения для стора

## 7) Сборка
- Создать `eas.json`
- Выполнить сборку preview и production
